# company

Монорепо ЕГЭ-чекера: общий код для Telegram-бота и сайта поверх одной базы данных.

## Структура

```
company/

├── core/                    # общий код
│   ├── db.py                # работа с Postgres
│   ├── schema.sql           # схема БД
│   ├── claude.py            # клиент Claude API
│   ├── grok.py              # клиент Grok API (распознавание рукописи)
│   └── config.py            # общие настройки (ключи, env и т.д.)
│
├── botback/                 # backend Telegram-бота
│
├── siteback/                # backend сайта
│
├── botfront/                # фронтенд Telegram Mini App (если будет)
│
└── sitefront/               # фронтенд сайта (Next.js/Vite и т.п.)
```

Оба бэкенда — отдельные службы на Railway из этого же репозитория, но общий код
(`core/`) один на двоих: правишь в одном месте — меняется у обоих, копии не разъезжаются.
Синхронизация бота и сайта идёт через общую базу.

## База данных

Единая Postgres-база. Пользователь опознаётся по внутреннему `users.id`, а входить
может по-разному: Telegram (общий аккаунт бота и сайта), email или Google (только сайт).
Публичный счётчик `counters.total_checks` — общее число проверок для главной страницы сайта.

Строка подключения берётся из переменной окружения `DATABASE_URL` (см. `.env.example`).

## Проверка базы

```bash
pip install -r requirements.txt
python -m core.db --init       # создать/обновить таблицы
python -m core.db --selftest   # быстрый прогон: юзер, баланс, история, счётчик
```
