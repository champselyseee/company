# ЕГЭ-чекер — фронтенд сайта (sitefront)

Веб-версия ЕГЭ-чекера: React + Vite + TypeScript. Деплой на Vercel.

Все обращения к серверу собраны в одном слое — `src/lib/api/`. Фронт не знает
про адреса и `fetch`, а только вызывает типизированные функции (`api.auth.me()`,
`api.checks.checkWork(...)` и т.д.). Бэкенд сайта (`siteback/`) подключается
одной переменной окружения `VITE_API_URL` — переписывать страницы не нужно.

## Переменные окружения

Скопируй `.env.example` в `.env` и укажи адрес бэкенда:

```
VITE_API_URL=https://адрес-бэкенда
```

Пока переменная пустая, сетевых запросов нет: страницы показывают аккуратные
пустые состояния (баланс и статистика — нули, история пустая). Примеры разбора
на главной («Посмотреть пример») работают всегда — это витрина.

## Локальный запуск

```bash
npm install
npm run dev          # http://localhost:5173
```

## Сборка и проверка типов

```bash
npm run build        # tsc -b + vite build → dist/
npm run typecheck    # только проверка типов
```

## Деплой на Vercel

Framework — Vite (определяется сам), Build Command `npm run build`, Output `dist`.
В настройках проекта пропиши переменную окружения `VITE_API_URL`. Конфиг —
`vercel.json` (SPA-переадресация на `index.html`).

## Что внутри

- `src/lib/api/` — слой API: `client` (база), `auth`, `checks`, `profile`,
  `billing`, `stats`, общие типы. Единая точка обращения к серверу.
- `src/components/pages/` — страницы: Проверка, База аргументов, Тарифы,
  Оформление, Проекты, Счётчик, Профиль, Вход.
- `src/lib/demo.ts` — примеры разбора для витрины «Посмотреть пример».
- `src/lib/billing.ts` — тарифы, пакеты и способы оплаты (статический конфиг).
- `src/lib/projects.ts` — список проектов (замени ссылки/лого на свои).
- `src/data/knowledgeBase.json` — база аргументов (зашита в сборку).

## Эндпоинты бэкенда, которые ждёт фронт

| Метод | Путь | Назначение |
|---|---|---|
| POST | `/api/auth/register` | регистрация по email |
| POST | `/api/auth/login` | вход по email |
| POST | `/api/auth/logout` | выход |
| GET  | `/api/me` | текущий пользователь и баланс |
| GET  | `/api/auth/{google,telegram}/start` | вход через сервис (редирект) |
| POST | `/api/auth/forgot` | восстановление пароля |
| POST | `/api/checks` | проверка работы → разбор |
| POST | `/api/ocr` | распознавание текста с фото |
| GET  | `/api/history` | история проверок |
| GET  | `/api/profile` | статистика и достижения |
| POST | `/api/payments` | создание платежа |
| GET  | `/api/public/total-checks` | публичный счётчик проверок |
